系统架构

五阶段自动化流水线

从 RSS 抓取到全球分发，每一步都经过精心设计与工程化落地

🌐
阶段一 · 资讯抓取与正文提取

并发解析 config/sources.yaml，提取 URL 后双引擎提取全文正文（优先 trafilatura，回退 readability-lxml），生成结构化 RawItem 列表。

httpx trafilatura readability-lxml sha256 唯一 ID
🔍
阶段二 · 三层精准去重

第一层：规范化 URL 绝对排重。第二层：RapidFuzz 标题相似度比对（阈值 92%）。第三层：Jieba 分词后关键词重合度检测，彻底消除重复信息。

RapidFuzz jieba 48h 滚动窗口
🧬
阶段三 · DBSCAN 聚类 + 五维打分

TF-IDF 字符级 N-Gram 提取特征，计算余弦相似度矩阵并输入 DBSCAN 聚类。每簇取全文最丰富的代表文章，再按社会热度、技术创新、信息量、受众相关、信源权威五维量化评分。

scikit-learn DBSCAN TF-IDF 5维评分 5-15分
🤖
阶段四 · LLM 双轨内容创作

精选事件簇送入大语言模型，同步生成两路内容：播客双人对话剧本（用于 TTS 合成），以及媒体编辑风格的 Markdown 文字日报（直接发布）。

OpenAI Gemini 讯飞星辰 MaaS 双轨并行
📡
阶段五 · 语音合成 + 发布分发

Edge-TTS 并行合成分段音轨，pydub 无缝拼接叠加 BGM，响度归一化后输出 MP3。同步生成符合 Apple Podcasts 规范的 feed.xml 与 index.html，GitHub Actions 自动部署至 GitHub Pages。

edge-tts pydub RSS 2.0 GitHub Actions GitHub Pages
快速开始

三步本地运行

推荐使用 uv 进行快速、稳定的依赖管理

Step 01
配置 API 密钥
# 在项目根目录创建 .env 文件
cp .env.example .env

# 填入你的 API Key
SPARK_API_KEY="your-spark-api-key"
Step 02
安装依赖
# 使用 uv（推荐）
git clone https://github.com/your/repo.git
cd ai-news-podcast
uv sync
Step 03A
生成今日播客（MP3 + RSS）
uv run podcast-daily --base-url http://localhost

# 无法连接 Edge-TTS 时追加 --no-audio
uv run podcast-daily --no-audio
Step 03B
生成文字日报（Markdown）
# 调用讯飞星辰 MaaS API 生成日报
uv run podcast-report

# 输出保存至 data/reports/ 目录
配置文件

灵活定制你的播客

通过 config/ 目录下的文件控制一切行为

文件 作用 关键配置项
config/config.yaml 核心行为控制文件 llm 模型选择、TTS 声线、抓取超时限制、关键词权重评分参数
config/sources.yaml 新闻源管理 精选 RSS/Atom 订阅源列表（Hacker News、各 AI 官方博客），支持单源启用/禁用
.env 密钥与私有配置 SPARK_API_KEY、OPENAI_API_KEY、GEMINI_API_KEY 等 API 密钥
.github/workflows/daily.yml CI/CD 自动化配置 定时触发 cron、环境变量注入、构建 + 部署 GitHub Pages 全流程
